## 1. 배경 및 목표

### 1.1 문제
사용자가 이사·여행 등으로 **지역별 분리수거 규칙이 다른 상황**에서, 급하게 버리려 할 때 “어디에 무엇을 어떻게” 버려야 할지 확신하기 어렵다. 기존의 텍스트 위주 검색은 직관성이 떨어지고 정보가 파편화되어 있다.

### 1.2 제품 비전
**실시간 AI Vision 분석 + 자체 730개 품목 DB + Supabase 전국 지자체 룰 + 단계별 픽토그램 가이드**를 한 화면에서 통합 제공하여, 텍스트 의존도를 낮추고 신뢰할 수 있는 로컬 정보로 올바른 배출 행동을 유도한다.

### 1.3 목표(Goals)
- **행정안전부(행안부) API 연동 및 Supabase 클라우드 DB 구축**: 전국 지자체의 최신 분리수거 규정과 행정동 정보를 실시간으로 수집·동기화하여 GPS 기반으로 사용자 위치에 맞는 배출 규칙을 노출한다.
- **자체 DB(730개 품목) 기반 고속 매칭**: 기존의 무거운 YOLO 모델을 제거하고, 자체 구축한 730개 품목 데이터베이스와 멀티모달 AI(Gemini 3.1 Flash Lite 등)를 결합하여 빠르고 정확한 사물 판별을 수행한다.
- **직관적인 사용자 인터랙션 및 시각 피드백**: AI 자동 인식(녹색 박스)뿐만 아니라, 사용자가 직접 화면을 터치하거나 드래그하여 영역을 지정할 때 즉각적으로 **주황색 네온 바운딩 박스(`NeonOrange`)**를 표시하여 사용자 주도적인 분석을 지원한다.
- **하단 슬라이딩 카드 고도화 및 핀치 줌 UX 최적화**: 분류 결과, 지자체 팁, 픽토그램 단계, 배출 요일뿐만 아니라, 핀치 줌 사용 시 하단 정보 카드 전체가 확대/축소되고 완벽하게 스크롤되도록 계층 구조를 최적화한다.
- **대형폐기물 및 폐가전제품 무상 수거 연계**: AI 분석 결과 폐가전제품으로 판별될 경우, 지자체 안내 다음 단계로 정부 운영 **'E-순환거버넌스' 무상 방문 수거 안내(단일/다량 수거 기준표)**를 자동으로 노출하고 관련 데이터를 DB에 연동한다.

### 1.4 비목표(Non-Goals)
- 일반 쓰레기 무단 투기 적발 및 지자체 과태료 부과 시스템과의 직접적인 행정 연동.
- 앱 내에서의 직접적인 폐기물 수거 기사 배차 및 결제 시스템 구축(E-순환거버넌스 등 외부 공식 채널 안내로 대체).

## 2. 사용자 및 시나리오

### 2.1 핵심 페르소나
`problem.md`에 정의된 **고양시 일산 등 낯선 지역으로 이사 온 60대 사용자** 및 일반 스마트폰 사용자: 스마트폰 기본 조작 가능, **급한 상황·큰 글자·직관적 그림 및 핀치 줌 확대** 선호.

### 2.2 주요 시나리오
1. **일반 재활용품 배출**: 분리수거함 앞에서 페트병을 들고 앱 실행 → 위치 자동 확인(행안부 API 연동) → 카메라 비춤 → AI와 자체 730개 DB 매칭 → 하단 카드에서 “비우기·라벨 제거·압착” 확인 후 배출.
2. **사용자 직접 영역 지정(Custom Drag)**: 여러 물건이 겹쳐 있거나 AI 자동 인식이 원하는 물체를 포착하지 못할 때, 사용자가 손가락으로 직접 화면에 대고 네모를 그림 → 해당 영역에 **주황색 네온 바운딩 박스** 생성 → "이 영역 분석" 팝업 터치 → 정밀 분석 진행.
3. **폐가전제품 배출 시나리오**: 안 쓰는 TV나 선풍기를 스캔 → AI가 '폐가전제품'으로 인식 → 지자체 대형폐기물 배출 안내와 함께 **'E-순환거버넌스 무상 방문 수거 안내표'** 노출 → 사용자가 무료 방문 수거 팁(단일 1개 가능, 소형 5개 이상 가능)을 확인하고 손쉽게 해결.
4. **대화형 클리어(Clarification)**: 인식이 불확실하거나 사용자가 결과를 거부할 때 → 바텀시트에 대화 패널과 후보군 칩이 열리고, 간단한 터치나 텍스트 입력으로 최종 품목을 확정.
5. **핀치 줌 확대 탐색**: 글씨가 작아 돋보기가 필요한 사용자가 하단 정보 카드를 두 손가락으로 펼침 → 카드 전체가 2배로 커지며, 아래로 스크롤다운해도 맨 밑의 법적 고지 및 저작권 정보까지 잘림 없이 완벽하게 스크롤됨.

## 4. UX / UI 스펙

### 4.1 핀치 줌(Pinch Zoom) 및 스크롤 계층 최적화
- **문제점 극복**: 기존에 핀치 줌 사용 시 내부 콘텐츠만 확대되어 아래쪽 내용이 잘리거나 스크롤이 안 되던 현상을 완벽히 해결.
- **구조적 스펙**:
  - 가장 바깥쪽 `Box`에서 `transformable`을 통해 두 손가락 핀치 줌(`scale`)을 감지.
  - 그 내부의 `Box`에 `verticalScroll`을 걸어 한 손가락 수직 스크롤을 통합 관리.
  - 실제 콘텐츠가 담긴 `Column`에 `Modifier.layout`을 적용하여, 컴포넌트가 부모에게 보고하는 실제 차지 높이를 `(원래 높이 × scale)`로 동적 확장.
  - `graphicsLayer`의 `transformOrigin`을 상단 중앙(`TransformOrigin(0.5f, 0f)`)으로 고정하여 깨짐이나 상단 밀림 없이 아래쪽으로 자연스럽게 확대.
  - 플로팅 화살표 버튼은 스크롤 뷰 바깥쪽 하단에 고정 배치.

### 4.2 디자인 토큰(Tokens.kt)
| 토큰명 | 색상값 / 속성 | 용도 및 설명 |
|--------|---------------|--------------|
| `RecycleGreen` | `#10B981` | 앱 전반의 친환경 재활용 아이콘 및 메인 브랜딩 포인트 컬러 |
| `RecycleGreenSoft`| `#ECFDF5` | 재활용 아이콘의 부드러운 배경 컨테이너 색상 |
| `NeonGreen` | `#7CFF6B` | AI 자동 포착 사물 Bounding Box 발광 컬러 |
| `NeonOrange` | `#FF6D00` | 사용자가 직접 탭/드래그하여 지정한 커스텀 영역 Bounding Box 발광 컬러 |
| `Primary` | `#1E293B` | 프리미엄 다크 네이비 메인 테마 (헤더, 버튼, 강조 텍스트) |
| `PrimarySoft` | `#F1F5F9` | 다크 네이비와 어울리는 부드러운 카드 및 배너 배경 |

### 4.3 타이포그래피 및 스타일
- **모던 산세리프**: 고해상도 환경에서 선명한 텍스트 렌더링. 복잡하고 딱딱한 법률 용어를 배제하고 직관적이고 친근한 사용자 어투 유지.
- **글래스모피즘**: 반투명 블러 효과와 슬릭한 네온 테두리를 결합하여 세련된 시각적 경험 제공.

## 5. 콘텐츠 및 데이터 아키텍처

```
+-----------------------------------------------------------------------------------+
|                        [ 정부 공공 데이터 & 클라우드 DB ]                            |
|                                                                                   |
|  1. 행정안전부 API (전국 지자체 행정구역 및 최신 분리수거 규정 자동 수집)               |
|  2. Supabase Cloud DB (전국 지자체 룰, 730개 품목 매핑 테이블, 사용자 피드백 저장)      |
+-----------------------------------------------------------------------------------+
                                         │
                                         ▼ (REST / SDK 동기화)
+-----------------------------------------------------------------------------------+
|                           [ 안드로이드 클라이언트 (앱) ]                             |
|                                                                                   |
|  • 자체 DB (730개 핵심 품목 로컬/캐시 데이터베이스 - 초고속 1차 필터링 및 매칭)          |
|  • Gemini 3.1 Flash Lite API (멀티모달 비전 분석 및 예외 상황 자연어 추론)             |
|  • E-순환거버넌스 무상 방문 수거 룰셋 (폐가전제품 전용 안내 모듈)                       |
+-----------------------------------------------------------------------------------+
```

- **행안부 API 연동**: 전국 200여 개 이상의 기초 지자체 분리수거 규정을 실시간으로 수집하여 Supabase DB에 적재.
- **Supabase DB**: 지자체별 분리배출 룰, 730개 품목 메타데이터, 사용자 정정 요청 로그 등을 통합 관리하는 클라우드 백엔드.
- **자체 730개 품목 DB**: 모바일 기기 내에서 즉각적인 매칭을 수행할 수 있도록 최적화된 핵심 품목 데이터베이스.

## 6. 기술 고려사항 (개발 핸드오프)

- **AI 비전 아키텍처 전환**: 기존 YOLO 모델의 가중치 파일 관리 및 무거운 온디바이스 추론 로직을 완전히 제거. 카메라 캡처 바이트 및 커스텀 드래그 영역 좌표를 자체 DB 키워드 매칭 및 Gemini 멀티모달 API로 전송하는 경량화 파이프라인 구축.
- **Supabase SDK 및 행안부 API 연동**: 클라이언트에서 위치 좌표 획득 시, 행안부 API를 통해 행정동을 추출하고 Supabase에 쿼리하여 해당 지자체의 룰(`ItemRule`)을 가져옴.
- **UI 상태 관리**: `AppState.kt`를 통해 카메라 캡처, 바운딩 박스 모드(AI vs Custom), 바텀시트 상태(`SheetState`), 핀치 줌 스케일 등을 깔끔한 단방향 데이터 흐름(UDF)으로 제어.

## 7. 성공 지표

- **이탈률 감소**: 초기 화면에서 직관적인 사용 가이드를 읽고 실제 사물 스캔 및 배출 안내까지 도달하는 비율 90% 이상 달성.
- **Custom Drag 활용률**: 복잡한 환경에서 사용자가 직접 주황색 바운딩 박스를 사용하여 정확한 분석을 이끌어내는 세션 비율.
- **폐가전 무상 수거 전환율**: 폐가전제품 스캔 시 E-순환거버넌스 안내를 통해 지자체 스티커 구매 대신 무상 방문 수거 혜택을 인지·활용하는 비율.
- **핀치 줌 만족도**: 고령층 및 저시력 사용자의 하단 정보 카드 스크롤 및 확대 탐색 만족도 95% 이상.

## 8. 리스크 및 미결정 사항

- **행안부 API 및 Gemini API Rate Limit**: 전국 단위 사용자가 몰릴 경우 공공 API 및 AI API의 일일 호출 한도 초과 위험.

- **Supabase 클라우드 동기화 지연**: 오프라인 또는 음영 지역에서의 DB 조회 지연.
