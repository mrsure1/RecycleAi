# 카메라 실시간 사물 인식 및 녹색 바운딩 박스 기능 기술 명세서

## 1. 개요 (Overview)
본 문서는 RecycleAI 애플리케이션의 카메라 프리뷰 화면에서 제공되는 **실시간 사물 인식 및 녹색 바운딩 박스(네온 테두리) 표시 기능**의 기술적 구현 방식과 시스템 아키텍처를 명시합니다.

사용자 경험(UX) 측면에서 스마트폰 카메라를 비추면 물체를 자동으로 인식하여 녹색 테두리가 따라다니기 때문에 스마트폰 운영체제(OS)의 기본 기능으로 오인될 수 있으나, 본 기능은 **앱 내부에서 자체 탑재한 모바일 AI 라이브러리와 맞춤형 그래픽 코드로 직접 구현한 독립적인 시스템**입니다.

---

## 2. 핵심 아키텍처 (System Architecture)

본 시스템은 실시간 성능(Low Latency)과 높은 인식 정확도(High Accuracy)를 동시에 달성하기 위해 **온디바이스 AI(Google ML Kit)**와 **클라우드 AI(Google Gemini AI)**를 결합한 하이브리드 파이프라인 구조를 채택하고 있습니다.

```
+-----------------------------------------------------------------------------------+
| 📱 스마트폰 카메라 (CameraX)                                                       |
|  - 실시간 프레임 스트림 (ImageProxy) 생성                                          |
+-----------------------------------------------------------------------------------+
                                         │
                                         ▼ [1. 실시간 스트림 전달]
+-----------------------------------------------------------------------------------+
| 🤖 Google ML Kit (온디바이스 AI / MlKitDetector.kt)                               |
|  - ObjectDetection (STREAM_MODE)                                                  |
|  - 사물의 구체적 품목 분류 없이 물체의 위치(Bounding Box) 좌표만 신속하게 감지      |
+-----------------------------------------------------------------------------------+
                                         │
                                         ▼ [2. 사물 영역 좌표 전달]
+-----------------------------------------------------------------------------------+
| 🎨 Jetpack Compose UI (커스텀 렌더링 / Overlay.kt)                                |
|  - Canvas 기반 커스텀 렌더링 (drawNeonBrackets)                                   |
|  - 네온 그린(#7CFF6B) 색상의 모서리 괄호 및 반투명 글로우 효과 표시                |
+-----------------------------------------------------------------------------------+
                                         │
                                         ▼ [3. 사용자가 바운딩 박스 터치 (Tap / Drag)]
+-----------------------------------------------------------------------------------+
| ☁️ Google Gemini AI (클라우드 AI / GeminiClient.kt)                                |
|  - gemini-3.1-flash-lite 모델 호출                                                |
|  - 터치된 영역의 이미지를 정밀 분석하여 한국어 분리수거 키워드 추출                |
+-----------------------------------------------------------------------------------+
                                         │
                                         ▼ [4. 키워드 기반 매칭]
+-----------------------------------------------------------------------------------+
| 🗄️ 앱 내부 DB 및 UI 안내 (InfoSheet.kt)                                           |
|  - 지자체 분리수거 품목 사전 DB와 일치하는 배출 가이드 제공                        |
+-----------------------------------------------------------------------------------+
```

## 3. 상세 동작 단계 및 주요 구현부

### 3.1. 1단계: 실시간 사물 위치 감지 (Google ML Kit)
* **주요 구현 파일:** `app/src/main/java/app/trashai/vision/MlKitDetector.kt`
* **기술 스택:** Google ML Kit Vision (`com.google.mlkit:object-detection`)
* **동작 원리:**
  * CameraX로부터 들어오는 실시간 이미지 프레임(`ImageProxy`)을 스트림 모드(`STREAM_MODE`)로 분석합니다.
  * ML Kit 내부의 경량화된 사물 감지 모델이 작동하여 화면 내에서 두드러진 피사체(Salient Object)들의 사각형 영역(`RectF`) 좌표를 추출합니다.
  * **설계적 특징:** 이 단계에서는 사물이 '페트병'인지 '캔'인지 분류(Classification)하지 않습니다. 과거 버전에서 ML Kit의 5개 카테고리 분류기가 부정확한 레이블을 제공했던 문제를 해결하기 위해, 현재는 오직 **위치 좌표(Bounding Box) 감지용**으로만 사용합니다.

### 3.2. 2단계: 실시간 바운딩 박스 렌더링 (Jetpack Compose UI)
* **주요 구현 파일:** `app/src/main/java/app/trashai/vision/Overlay.kt`
* **기술 스택:** Jetpack Compose Canvas, Pointer Input
* **동작 원리:**
  * `MlKitDetector`가 반환한 원본 이미지 기준의 사물 좌표를 현재 스마트폰 화면 크기 및 비율에 맞게 변환(`mapAll`)합니다.
  * Compose의 `Canvas` 위에서 `drawNeonBrackets` 함수를 호출하여 사물의 모서리를 감싸는 네온 테두리를 렌더링합니다.
  * **그래픽 효과:** 가시성과 세련된 사용자 경험을 위해 메인 색상(`NeonGreen`, `#7CFF6B`), 반투명 글로우 효과, 중심부 하이라이트(화이트)를 겹쳐 그려 역동적이고 미래지향적인 UI를 제공합니다.

### 3.3. 3단계: 사용자 상호작용 및 정밀 AI 분석 (Google Gemini AI)
* **주요 구현 파일:** `app/src/main/java/app/trashai/gemini/GeminiClient.kt`
* **기술 스택:** OkHttp3, Google Gemini API (`gemini-3.1-flash-lite`)
* **동작 원리:**
  * 사용자가 화면에 표시된 녹색 바운딩 박스를 터치(Tap)하거나 원하는 영역을 드래그하여 선택하면, 해당 영역의 이미지가 Base64 형태로 인코딩되어 구글 클라우드의 Gemini AI 서버로 전송됩니다.
  * Gemini 모델은 정교한 멀티모달 비전 분석을 수행하여 사진 속 피사체의 핵심 품목을 식별하고, 지자체 품목 사전(`wasteguide.or.kr`)에서 사용되는 표준 한국어 키워드(예: `투명페트병`, `종이팩`, `건전지` 등)를 JSON 형태로 반환합니다.

## 4. 설계 결정 사항 및 기대 효과 (Design Decisions & Benefits)

| 구분 | 채택 기술 | 채택 사유 및 기대 효과 |
| :--- | :--- | :--- |
| **위치 감지** | **Google ML Kit (온디바이스)** | 클라우드 통신 없이 기기 내부에서 실시간(60fps 급)으로 동작하므로, 지연 시간(Latency) 없이 부드럽게 사물을 따라다니는 UI 구현 가능 |
| **품목 식별** | **Google Gemini AI (클라우드)** | 복잡하고 다양한 형태의 쓰레기를 정확하게 판단하기 위해 대규모 시각 언어 모델(VLM)을 활용하여 인식 정확도를 극대화 |
| **UI 렌더링** | **Jetpack Compose Canvas** | 안드로이드 OS 기본 카메라 UI에 종속되지 않고, 브랜드 아이덴티티에 맞춘 독창적인 네온 그린 테두리와 애니메이션 효과를 유연하게 제어 가능 |

---

## 5. 요약 (Summary)

RecycleAI의 녹색 바운딩 박스 기능은 **안드로이드 스마트폰의 기본 기능이 아니며**, 기기 내부의 빠른 사물 추적 AI(ML Kit)와 클라우드의 강력한 비전 분석 AI(Gemini), 그리고 맞춤형 UI 렌더링 기술이 유기적으로 결합되어 탄생한 고도화된 기능입니다.
