19 Correction

중간발표 정정 — Supabase는 런타임 검색 DB가 아닙니다

수정·추가 하이라이트: 현재 앱은 Supabase에 연결해서 실시간 사물검색을 하지 않습니다

오해: Supabase 실시간 검색

중간발표에서는 앱이 Supabase에 연결되어 사물을 실시간 검색하는 것처럼 전달되었습니다.

정정: 앱 안 SQLite 우선

현재 앱 런타임에는 Supabase·PostgREST·백엔드가 없고, 품목·조례·MOIS·문의처를 로컬 SQLite에서 조회합니다.

Gemini의 실제 역할

Gemini는 애매한 사진을 한글 키워드로 바꾸는 보조 역할만 합니다. 사용자에게 보이는 안내문은 공공 DB 문구입니다.

Supabase의 향후 사용 가능성

행안부 품목 DB가 730여 개에서 1500개로 확장될 때, 앱 자체 SQLite 업데이트 작업에만 임시 도구로 검토합니다.

발표 멘트: “Supabase는 현재 앱에서 실시간 검색 DB로 쓰지 않습니다. 배포 전 데이터 정제·로컬 DB 재생성 단계에서만 임시 활용할 수 있습니다.”
20 Eval

Eval 테스트 케이스 5개

점수가 낮았던 항목은 수정 방향까지 함께 기록

평가 항목 테스트 기준 낮은 점수 원인 수정 내용
핀치 줌 확대/축소 후 하단 고지까지 스크롤 확대 시 하단이 잘림 layout 기반 줌 컬럼으로 전체 스크롤
Supabase 배제 비행기 모드에서도 규칙 카드 표시 클라우드 검색처럼 설명됨 SQLite 번들 + Gemini 보조로 명확화
DB 통합 품목·지역·MOIS·문의처가 한 카드에 결합 지역별 안내가 분산됨 통합 SQLite에서 지역 기준 매칭
반복 안내문 동일 주의문은 1회만 노출 세척·라벨 안내가 반복됨 공통 가이드를 1회 카드로 단순화
오터치 방어 작은 드래그·빈 화면 탭에서 복구 작은 박스 후 UX가 막힘 32px 이하 취소, 배경 탭 해제
