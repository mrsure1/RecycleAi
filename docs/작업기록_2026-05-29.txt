(1) 핀치줌·스크롤 개선

  핀치줌이 즉각 반응하지 않던 문제 수정.
- 한 손가락 이동(스크롤)과 두 손가락 확대/이동(핀치줌) 충돌 해결.
  · 줌 안 한 상태: 세로 스크롤 정상 작동
  · 줌 한 상태: 손가락 이동 거리만큼 자연스럽게 패닝
- 첫 핀치 시 한 단계만 확대되고 멈추던 버그 수정
  (pointerInput key에서 동적 콘텐츠 크기 제거 → 제스처 중단 방지).
- 꼬이고 불필요해진 코드 정리/삭제.
- 파일: ui/PinchZoomScrollColumn.kt, MainActivity.kt, ui/CardSheet.kt

(2) 결과 카드 UI 개선
- "주의" 섹션: 내용이 비어 있으면(null/공백) 아예 표시하지 않도록 처리.
- 지역 "배출시간/배출장소"를 그 아래 관련부처 연락처보다 더 눈에 띄게
  시인성 조정(글자 크기·굵기·색·배경/보더).
- 관련부처명+전화번호를 얇은 보더선으로 한 덩어리로 묶어 여백 축소.

(3) 전화 통화 후 상태 유지
- 결과 화면에서 전화번호를 눌러 통화 후 돌아오면 초기화면으로
  리셋되던 문제 수정.
- 전화 연결로 앱이 잠시 멈출 때만(keepResultOnNextStop 플래그)
  결과 화면을 그대로 유지, 그 외 상황은 기존대로 초기화.

(4) 무료 분석 횟수 초과 안내(AdLimitReached) UI
- 일반 안내문과 똑같아 와닿지 않던 문제 → 확실히 구분되는
  "충전 팝업형" 카드로 디자인 개편(경고 배지/잠금 아이콘/통계 행/버튼).
- 횟수 초과 메시지를 보여줄 때는 결과 화면처럼 시트를 위로 올려
  한 화면에 모든 내용·버튼이 보이도록 조정.
- 광고 시청(또는 충전) 후에는 카메라 초기화면으로 복귀하도록 통일.

(5) 광고 — 배너 + 보상형 실제 연동  ★ 오늘 후반 핵심
- 광고 ID 중앙 관리 파일 신설: ads/AdIds.kt
  · debug 빌드 → Google 테스트 ID 자동 사용(계정 정지 방지)
  · release 빌드 → 실제 ID(PROD_BANNER / PROD_REWARDED) 사용
- 보상형 광고를 실제 AdMob RewardedAd SDK로 구현: ads/RewardedAdManager.kt
  · preload(미리 로드) / showWhenReady(준비되면 표시, 로딩 중이면 자동 표시)
  · 영상 끝까지 시청 완료 시에만 충전(onReward)
  · 미로드·중도이탈 시 충전 안 함 + 안내 문구
- "광고 보고 N회 충전하기" 버튼을 가짜 카운트다운 → 실제 광고 호출로 교체.
- 배너 광고 ID도 하드코딩 → AdIds.banner 사용으로 변경.
- 파일: ads/AdIds.kt(신규), ads/RewardedAdManager.kt(신규),
        MainActivity.kt, ui/CardSheet.kt
- 남은 일: AdMob 콘솔에서 배너/보상형 광고단위 ID 발급 후
          AdIds.kt의 PROD_* 두 줄 교체. (앱 등록·광고단위 생성은 콘솔 수동 작업)
